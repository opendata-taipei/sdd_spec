# Feature Specifications

本目錄追蹤跨版本、長期存在的產品能力。Feature 文件回答「要提供什麼能力與價值」；`changes/` 則記錄某次實作、修正或發布如何通過 SDD lifecycle。

## ID 與狀態

- Feature ID：`FEAT-NNN`
- Status：`Proposed`、`Planned`、`In Progress`、`Available`、`Deprecated`、`Retired`
- 每個 Feature 至少連結一個 Change ID，並保留版本與狀態歷史。

## 必要欄位

- Feature ID、名稱、狀態、Owner、Target Release
- 問題、使用者價值、In Scope／Out of Scope
- Acceptance Outcomes、Security／Privacy boundaries
- Related Changes、Dependencies、Risks、Release History

Feature 文件不得取代 Change Requirements、Design、Test Plan、Human Gate 或 Evidence。
