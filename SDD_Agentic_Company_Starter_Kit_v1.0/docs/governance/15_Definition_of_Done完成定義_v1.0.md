# SDD-GOV-015｜Definition of Done（DoD）

> 文件狀態：草案（Draft）  
> 版本：1.0  
> 文件擁有者：[公司名稱／部門]

## Done 層級

Task Done、Change Done、Release Done、Closure Done。

## 核心原則

Done 必須有可查核證據；AI 宣稱、PR 合併或 Pipeline Success 均不足以單獨代表完成。

## 必要條件

Requirements 驗證、Review／Test／Security 通過、API／Data／Migration／Rollback 完成、監控正常、Living Spec／Drift／Archive 完成。

## 禁止宣告 Done

未完成測試／文件、未核准 Scope、Unspeced Drift、無法解釋 AI 產出、未完成觀察期、未追蹤 TODO 或技術債。
