## 規範用語

SHALL＝強制；SHALL NOT＝禁止；SHOULD＝建議且需例外理由；MAY＝允許。

## 需求欄位

- Requirement ID
- 標題／優先級／來源
- EARS／SHALL 需求敘述
- 驗收條件
- 錯誤、邊界與權限情境
- 相依需求與 Test ID
- 狀態與 Owner

## 可攜性檢查

需求不得依賴特定 AI 聊天脈絡；不同 AI 應能解析相同範圍、行為、限制與驗收條件。
